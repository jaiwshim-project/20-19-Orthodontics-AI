User Manual

사용자 매뉴얼

20-19 Orthodontics AI 플랫폼의 모든 기능을 단계별로 설명합니다.

1. 시작하기 — 환자 등록

플랫폼의 모든 진단은 현재 활성 환자에 종속됩니다. 좌측 사이드바 또는 상단 우측의 "환자 변경" 버튼으로 환자를 입력하세요.

입력 항목

  • 이름 — 식별용. 익명화 시 ID로 대체 가능.
  • 생년월일 — 자동으로 만 나이를 계산합니다.
  • 연령 구분 — 자동(만 17세 이하 어린이) 또는 수동 지정.
  • 성별 — 성장 예측 시 mid-parental 계산에 사용.
💡 환자 정보는 localStorage에 저장되며, 별도의 서버 동기화는 SaaS 플랜에서 활성화됩니다.

2. 3D 뷰어 사용법

Three.js 기반 STL 뷰어. 드래그앤드롭 또는 클릭으로 STL을 업로드하면 자동으로 EZL-STL 엔진이 측정값을 산출합니다.

지원 측정값

  • 치열궁 길이 (Arch Length, mm) — 좌측 부조화 분석에 사용
  • 치아 너비 합 (Sum of MD widths, mm)
  • 부조화 (Discrepancy = Arch − Tooth)
  • Bolton 비율 (전치 77.2%±2, 전체 91.3%±2)
  • Spee 만곡 (Curve of Spee, mm)
  • Crowding 분류 (Mild/Moderate/Severe)

조작법

  • 마우스 드래그 — 회전
  • 휠 — 줌
  • Shift + 드래그 — 패닝
  • 상단 툴바 — 정면/상단/측면/3D 프리셋, 와이어프레임 토글, PNG 스크린샷
⚠️ STL 파일은 50MB 이내로 제한됩니다. 더 큰 파일은 데시메이션 후 업로드하세요.

3. 발치 판단 AI

Tweed/Steiner/Down 분석을 통합한 룰베이스 사전 점수 + Gemini 정밀 분석.

핵심 입력값

  • ANB — 골격 관계 (정상 2°±2)
  • Crowding — 4mm 미만 비발치 우선, 8mm 이상 발치 강력 권장
  • FMA / IMPA — Tweed 분석 핵심
  • Profile / Lip strain — 안모 영향 평가

성인/어린이 분기

어린이 환자는 자동으로 -25점 가중치가 적용되어 비발치/2단계 치료가 우선 권장됩니다. 골성숙이 진행 중일 때는 발치보다 성장 활용이 효과적입니다.

💡 AI 분석 결과의 신뢰도가 70% 미만이면 추가 검사(CBCT, 사진 분석)를 권장합니다.

4. 성장 예측 AI

CVMS 단계 + 부모 키 + 골연령으로 잔여 성장량과 peak velocity 시점을 추정합니다. 어린이 전용.

CVMS 단계 가이드

  • CS1 — 사춘기 가속 전 (8-9세)
  • CS2 — 가속 시작 (9-10세)
  • CS3 — Peak velocity ★ 골격 치료 골든타임
  • CS4 — Peak 직후 (12-13세)
  • CS5 — 감속 (14-15세)
  • CS6 — 성장 완료 (16세 이상)

차트의 주황 점선이 peak velocity 시점이며, 이 구간에 기능 장치(Twin Block, Headgear 등)를 적용하는 것이 가장 효과적입니다.

5. 안모 시뮬레이션

측면 안모 SVG에 5개 슬라이더(상악 후방, 하악 전후방, 상순/하순, 턱)를 조정하면 실시간으로 프로파일이 변형됩니다.

핵심 지표

  • E-line (Ricketts) — 코끝(NoseTip)과 턱끝(Pog)을 잇는 미적 기준선. 상순은 -4mm, 하순은 -2mm 후방이 이상적.
  • 코-턱 비율 — 코 길이 / 하안면부 높이. 0.45-0.55 범위가 균형.

"📷 스냅샷" 버튼으로 변형된 프로파일을 PNG로 저장하여 환자 상담에 활용할 수 있습니다.

6. 재발 예측 AI

치료 종료 시점의 IMPA, 보정 장치, 환자 협조도 등을 입력하면 1·3·5·10년 시점의 재발 확률을 산출합니다.

위험도별 권장 프로토콜

  • 저위험 (<30%) — Essix 24개월 야간 착용 후 점진적 중단.
  • 중위험 (30-50%) — Essix + 1년 정기 검진.
  • 고위험 (>50%) — Bonded 3-3 + 야간 Essix + 6개월 검진.

7. RAG 챗봇 활용 팁

Gemini 1.5/2.0 + Supabase pgvector 기반 검색증강생성. 답변에는 참조한 출처가 우측 패널에 함께 표시됩니다.

좋은 질문 예시

  • "성인 환자 Crowding 5mm, 발치 vs 비발치 판단 기준은?"
  • "Bolton 비율 91.3%가 의미하는 바와 임상적 활용을 설명해줘"
  • "CVMS 3단계 어린이의 권장 치료 시기는?"

파일 첨부

구강 사진(JPG/PNG)을 첨부하면 Gemini Vision이 5가지 임상적 특징을 추출합니다. STL 첨부 시 메타데이터(삼각형 수, 크기)가 자동 추출됩니다.

8. FAQ

Q1. 진단 결과를 의료 기록으로 바로 사용할 수 있나요?
A. 본 시스템은 임상 의사결정 보조 도구입니다. 최종 진단은 반드시 자격을 갖춘 전문의의 판단에 따르십시오.
Q2. 환자 데이터는 어디에 저장되나요?
A. MVP는 브라우저 LocalStorage. SaaS 플랜에선 Supabase Postgres에 RLS(Row-Level Security) + 병원별 격리로 저장됩니다.
Q3. 외부 인터넷 없이도 작동하나요?
A. 정적 페이지(3D 뷰어, 룰베이스 분석)는 오프라인 가능. Gemini AI 호출과 RAG는 인터넷이 필요합니다.
Q4. STL 파일이 안 열립니다.
A. ASCII/Binary 모두 지원하지만 50MB를 초과하거나 손상된 파일은 거부됩니다. CloudCompare/Meshmixer로 데시메이션 후 다시 시도하세요.
Q5. Gemini 모델을 바꾸려면?
A. 챗봇 우측 상단 모델 셀렉터에서 Flash(빠름) / Pro(정밀) 선택. 백엔드는 환경변수 GEMINI_API_KEY 하나로 모든 모델 호출이 가능합니다.
Q6. 어린이 환자에게 발치 권장이 너무 보수적이지 않나요?
A. 의도된 동작입니다. 성장 잠재력을 활용한 비발치/2단계 치료가 장기적으로 더 안정적이라는 임상 컨센서스를 반영했습니다(가중치 -25).
Q7. 한글 인코딩 깨짐 문제가 발생합니다.
A. Vercel API는 latin1 → UTF-8 자동 변환을 적용합니다. 그래도 깨진다면 브라우저의 Content-Type 헤더를 확인하세요.
Q8. 다른 분석 시스템과 데이터 교환이 가능한가요?
A. CSV/JSON 내보내기는 대시보드에서 즉시 가능합니다. DICOM/HL7 FHIR 게이트웨이는 SaaS Pro 이상 플랜에서 제공됩니다.
Q9. 진단 결과를 PDF로 출력하고 싶습니다.
A. 각 진단 페이지에서 브라우저 인쇄(Ctrl+P) → PDF 저장. 향후 전용 PDF 리포트 생성 기능이 추가됩니다.
Q10. 의료기기 인증을 받았나요?
A. MVP는 임상 의사결정 보조 도구로 의료기기 등급에 해당하지 않습니다. SaaS GA 단계에서 식약처/FDA Class II SaMD 인증을 추진합니다.
